# INVESTMENT_POLICY.md - 投资决策标准

**生效日期**: 2026-02-10  
**适用范围**: Friday 模拟投资组合 (后续将扩展至真实资金账户)

---

## 一、核心原则

### 1.1 信誉度优先
- **绝不编造数据** - 所有股价、财报数据必须来自可靠 API 或官方披露
- **逻辑可验证** - 每个投资建议必须有完整、可追溯的分析链条
- **风险优先** - 先考虑"会亏多少"，再考虑"能赚多少"

### 1.2 人工确认机制
| 操作类型 | 阈值 | 要求 |
|----------|------|------|
| 新建仓位 | 单票 >5% | 需用户确认 |
| 加仓 | 单票 >10% | 需用户确认 |
| 重大调仓 | 单票 >20% 或 总仓位变动 >30% | 必须用户确认 |
| 止损/止盈 | 任意比例 | 自动执行，事后通知 |

## 二、AI 模型配置

### 2.1 投资决策任务

| 场景 | 主模型 | 推理级别 | Fallback | Fallback 推理 | 超时 |
|------|--------|----------|----------|---------------|------|
| **投资调仓决策** | `opus` (Claude 4.6) | medium | `kimi-coding/k2p5` | high | 600s |
| **紧急风控操作** | `kimi-coding/k2p5` | medium | - | - | 120s |
| **定期持仓检查** | `kimi-coding/k2p5` | medium | - | - | 300s |
| **标的深度分析** | `opus` | medium | `kimi-coding/k2p5` | high | 600s |

### 2.2 配置逻辑

**为什么 Opus + Medium 足够？**
- Opus 基础推理能力极强，medium 推理已能处理复杂投资分析
- 节省 token，提高响应速度
- 超长上下文支持完整财报、多份研报同时分析

**为什么 Kimi Fallback 需要 High？**
- 作为降级方案，需要更高推理级别补偿模型差距
- 确保降级后输出质量不大幅下降
- High 推理下的 Kimi 也能产出合格的投资分析

### 2.3 Fallback 执行流程

```
投资分析任务触发
    ↓
尝试调用 Opus (medium)
    ↓
├─ 成功 → 输出完整投资建议
│
└─ 失败 (额度/网络/错误)
    ↓
    自动降级到 Kimi (high)
        ↓
    输出投资建议 + 标注"[使用备用模型]"
    建议用户关注分析质量
```

## 三、分析框架标准

### 3.1 调仓决策必须包含

**模型信息记录：**
- 分析所用模型 (如: opus / kimi-coding/k2p5)
- 推理级别 (low/medium/high)
- 是否使用 Fallback (是/否)

示例格式：
```
[模型信息]
主模型: opus (medium)
Fallback: 否
```

#### 买入/加仓分析
1. **估值分析**
   - 当前 PE/PB/PS vs 历史均值
   - vs 同业对比
   - vs 市场基准 (S&P 500)

2. **增长逻辑**
   - 未来 12-18 个月催化因素
   - 利润增长驱动 (收入/利润率)
   - 叙事/预期差分析

3. **风险评估**
   - 主要风险点 (宏观/行业/公司)
   - 最坏情景下的下跌空间
   - 止损位设置

4. **仓位建议**
   - 建议仓位比例
   - 入场区间 (分档)
   - 目标价/止盈位

#### 卖出/减仓分析
1. 卖出理由 (估值到位/逻辑破坏/风险上升)
2. 剩余仓位建议
3. 替代标的 (如有)

### 3.2 戴维斯双击标的挖掘 (特殊任务)

**双重要求：**
- 利润增长 (EPS 增速 >20%)
- 估值提升 (PE 扩张空间 >30%)

**分析深度：**
- 价格回落原因 (暂时性 vs 结构性)
- 利润增长驱动 (收入/利润率/回购)
- 估值提升催化剂 (叙事转变/市场对标)
- 双击触发条件和时间框架

## 四、风险控制红线

### 4.1 禁止操作
- ❌ 单票仓位 >30% (除非用户明确授权)
- ❌ 单日调仓 >50% 总仓位
- ❌ 使用未验证的数据来源
- ❌ 在 API 失败时"估算"数据

### 4.2 必须触发警报的情况
- 持仓股单日跌幅 >10%
- 重仓股财报不及预期 (>20% 偏差)
- 宏观重大变化 (政策/地缘政治)
- 组合净值回撤 >15% 从高点

### 4.3 API 失败处理
1. **数据获取失败** - 立即停止分析，向用户求助
2. **模型调用失败** - 按 Fallback 流程切换模型
3. **推送失败** - 重试 3 次，失败则记录日志

## 五、报告质量标准

### 5.1 晨报质量检查清单
- [ ] VIX、恐惧贪婪指数数据来源明确
- [ ] 美股/港股大盘数据一致性检查
- [ ] MAG7 七只股票均有动态
- [ ] 重仓股 5 只全部覆盖
- [ ] 投资逻辑 5 要素齐全
- [ ] **模型信息已记录** (model: xxx, reasoning: xxx)
- [ ] 无明显语法或数据错误

### 5.2 调仓建议质量检查
- [ ] **模型信息已记录** (主模型/Fallback/推理级别)
- [ ] 买入理由 >= 3 个独立逻辑
- [ ] 风险分析 >= 2 个主要风险点
- [ ] 估值数据有明确来源
- [ ] 止损位设置在技术面/基本面关键位置
- [ ] 仓位建议与风险等级匹配
